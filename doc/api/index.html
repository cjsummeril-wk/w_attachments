w_attachments

A module that can render attachments inside a Virtual List.

Consuming

  • Add w_attachments to your pubspec.
  • Include the following stylesheets in your html:
    <link rel="stylesheet" href="packages/web_skin/dist/css/web-skin.min.css">
    <link rel="stylesheet" href="packages/web_skin/dist/css/peripherals/grid-v2.min.css">
    <link rel="stylesheet" href="packages/w_attachments/style/w_attachment.css">
  • Use w_module to contruct and load a AttachmentsModule.
class BaseModule extends Module {
  AttachmentsModule module;
  BaseModule() {
    module = new AttachmentsModule();
  }

  @override
  onLoad() async {
    loadChildModule(module)
  }
}

  • Render the module's content using react. The content must be rendered inside a web_skin_dart Flexbox Grid VBlock.

    import 'package:react/react_dom.dart' as react_dom;
    ...
    Element container;
    ...

    var attachmentsContent = (VBlock()..isNested = false)(module.components.content());
    react_dom.render(attachmentsContent, container);

Building

Install the Dart SDK

$ brew tap dart-lang/dart
$ brew install dart --with-content-shell --with-dartium
$ brew linkapps dart

Install SASS (for compiling CSS)

sudo gem install sass

Installing / Updating Dart Dependencies

Run the following from the root w_attachments directory.

pub get

Running

  • Run the example application inside the example directory to see a basic implementation of w_attachments

Running tests

  • run pub run dart_dev test in the root or app directory to run all the tests at those levels

  • run pub run dart_dev test -n REGEX to tests matching a specific REGEX. It's recommended to wrap all tests in a file with a corresponding top level group so that all tests in file can be run easily. - Ex. All tests in attachments_module_test.dart should be wrapped in a AttachmentsModule group so they can be run

    with `pub run dart_dev test -n AttachmentsModule` (or better yet: `pub run dart_dev test -n '^AttachmentsModule'`.
Make sure to wrap your regex in quotes if you are using any special characters - if your terminal runs caret substitution
or any other special zsh commands you may have unexpected results.)

Generating Tests

  • run pub run dart_dev gen-test-runner in the root directory.

Generating CSS

  • run ./tool/build_css.sh in the root directory after making changes to sass files.

Contributing

  • Format code using dartformat with 100 characters: pub run dart_dev format
