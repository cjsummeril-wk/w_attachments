A data provider for w_attachments to load attachments.

Extend this, provide a AttachmentsLoaderFactory to AttachmentsModule that constructs your implementation of AttachmentsLoader, and call HistoryModule.api.loadHistory(attachmentIdentifier) to tell w_attachments to create your loader and start asking it for attachment corresponding to the provided attachmentIdentifier and start populating the attachment list with this information.

attachmentsIdentifier → String

The attachmentsIdentifier correlates this AttachmentsLoader to a specific list of attachment items. This will be the same string passed into the attachments module loadHistory API call.

hasMoreAttachments → bool

Tells the attachment module if there are more attachments that can be loaded from this AttachmentsLoader. If true, then scrolling near the bottom of the attachments list can trigger the loading UI as well as fetchMoreAttachments to be called. If false, then fetchMoreAttachments will not be called again.

valid → bool

Whether or not the data coming from this AttachmentsLoader is valid or not. Whenever loadHistory is called on the attachments module API, the previous AttachmentsLoader will be invalidated so w_attachments knows not to display any stale data from said invalidated AttachmentsLoader
